Problems

If you receive an out of memory error from JavaPEG when running the application. Then it means that the processes that JavaPEG runs demand more memory than what currently is available for the Java Virtual Machine. There are several ways of solving this problem and two of them are described below:

  1. Edit the shortcut that is used to start JavaPEG and change the value of the parameter -Xmx to a larger value.
  2. If JavaPEG is started from a command line or a script: Then increase the -Xmx parameter.
