Unbuffered Logging:
All logging is instantly written to the log file and is not being buffered in the memory. This might be useful for debugging.

Rotate Log Automatically:
Means that the log file will be rotated, in other words that the current file will be closed and a new will be created automatically.

Compress Rotated Log File:
When a log file is rotated it is possible to have it compressed to save space. The compression format used is ZIP.

Log Rotate Size:
Defines at which size the log file will be rotated. The size is set either in KiB or in MiB. Smallest allowed size is 10 KiB and largest size is 100 MiB.

Log Level:
Specifies how detailed the logging will be. Possible alternatives are: DEBUG, INFO, WARN, ERROR and FATAL. DEBUG will generate most detailed logging and FATAL least detailed.

Log Name:
The name that the log file will have.

Log Entry Timestamp Format:
Specifies what the timestamp in the log file will look like.

Log Entry Timestamp Preview:
Displays a preview of what a log entry timestamp will look like in the log file.
